# Methodology

The working method this assistant encodes — developed across a real job
search, not theorized. Adopt what fits; the templates assume these
habits but don't require them.

## Context MDs as session memory

AI sessions end; Markdown files don't. Every durable decision, story,
and draft lives in a file, not a chat transcript. Any session — any
model — starts by reading the context files and continues where the
last one stopped. The files are the memory; the AI is the collaborator
of the moment.

## Excavation interviews, not cold drafting

The worst career materials come from an AI drafting cold off a resume.
The best come from being interviewed: the AI asks about a project until
the real story surfaces — what was at stake, what you actually decided,
what changed because of it. Talk out loud first, write second. The
career narrative template is built to receive what excavation produces.

## The "we" framing

Phrase the collaboration as *we* from the start: "what are we trying to
say here," "what's our angle on this role." It sounds small. It isn't —
it keeps the human inside the work instead of reviewing the AI's work,
and the output stays in the human's voice because the human never left.

## Modular, atomic career content

One source of truth (the career narrative), distributed everywhere else.
Resume bullets, LinkedIn sections, portfolio case studies, and cover
letter paragraphs are views of the same atomic stories — written once,
adapted per surface. When a story improves, every surface inherits the
improvement.

## Designed for handoff

Same principle as a production design system: session-portable,
AI-resumable, built for whoever (or whatever) picks it up next. If a
file only makes sense to the AI that wrote it, it's written wrong.
